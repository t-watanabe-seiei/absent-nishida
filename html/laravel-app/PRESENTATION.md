# 欠席連絡システム プレゼン資料

> 発表時間の目安：約5分

---

## 1. 従来の運用とその課題

### 電話連絡のアナログ方式

これまでの欠席連絡は、以下の流れで行われていました。

```
【従来の流れ】

保護者
  │  　電話
  ▼
事務室の職員
  │  手書きメモ
  ▼
担任教師（朝のホームルーム前に届けば間に合う）
```

### 具体的な問題点

| # | 問題 | 内容 |
|---|---|---|
| 1 | **受付時間が限られている** | 電話受付は **毎朝 8:00〜8:30 の30分間のみ** |
| 2 | **事務室への集中** | 同時間帯に電話が殺到し、つながらないことがある |
| 3 | **アナログな転達** | 職員が手書きメモを取り、担任に直接届けるまで時間ロス |
| 4 | **転達ミス・読み間違い** | 手書きメモの誤読、渡し忘れが起きる可能性がある |
| 5 | **集計が大変** | 月次・年次の欠席状況を集計するには、紙やExcelの手作業が必要 |
| 6 | **保護者が記録を持てない** | 連絡した内容を後から確認する手段がない |

> **「8:00〜8:30に電話できなかったら連絡できない」「電話がつながらない」**
> — 保護者・学校双方にとってのストレス源でした。

---

## 2. このシステムが解決すること

### 新しい欠席連絡の流れ

```
【新しい流れ】

保護者（スマホ・PCから24時間いつでも）
  │  　オンライン入力
  ▼
システム（自動処理）
  │  　自動メール通知（即時）
  ▼
担任教師（ログインしていつでも確認可）
                ↓
     管理者（スーパー管理者）が全体を管理
```

### 従来との対比

| 項目 | **従来（電話）** | **新システム** |
|---|---|---|
| 受付時間 | **8:00〜8:30 のみ** | **24時間・365日** |
| 連絡手段 | 電話（事務室経由） | スマホ・PCから直接入力 |
| 担任への伝達 | 手書きメモを手渡し | **自動メール通知（即時）** |
| 伝達ミスのリスク | メモの読み間違い・渡し忘れ | **なし（デジタル記録）** |
| 過去の連絡確認 | 困難（紙の保管） | **一覧でいつでも確認可** |
| 集計・統計 | 手作業（Excel等） | **リアルタイム自動集計** |
| 一括登録 | 手入力 | **CSVインポートで効率化** |

## 4. 利用者とロール

3種類のユーザーがシステムを利用します。

| ユーザー | できること |
|---|---|
| **スーパー管理者** | 全クラス・全データへのアクセス、システム全体の管理 |
| **担任教師** | 担当クラスのデータのみ管理・閲覧 |
| **保護者** | 自分の子の欠席連絡の登録・確認 |
| **生徒** | Classroomアドレスで保護者アカウントへ誘導 |

## 5. 主要機能

### 5-1. 保護者機能

- **欠席連絡の登録**：欠席 or 遅刻、理由、登校予定時刻を入力（24時間受付）
- **一覧・確認**：過去の欠席連絡を一覧で確認
- **編集・取り消し**：登録済みの連絡を修正・取り消し可能
- **パスワード変更**：自分のパスワードをいつでも変更可能

### 5-2. ダッシュボード（管理者）

管理者がログインするとリアルタイムで確認できます。

| 表示項目 | スーパー管理者 | 担任 |
|---|---|---|
| クラス数 | 全体 | 担当クラスのみ |
| 生徒数 | 全体 | 担当クラスのみ |
| 保護者数 | 全体 | 担当クラスのみ |
| 本日の欠席数 | 全体 | 担当クラスのみ |
| 本日の遅刻数 | 全体 | 担当クラスのみ |

### 5-3. 欠席連絡の管理・集計（管理者）

- **当日の欠席一覧**：当日の欠席・遅刻をすぐに確認
- **期間・クラス・区分でフィルター**：絞り込んで一覧表示
- **統計情報**：今日 / 今週 / 今月 / 累計の件数を表示
- **月別トレンド**：過去6ヶ月分の欠席数グラフ

### 5-4. マスタ管理（管理者）

- **クラス管理**：クラス情報（クラス名・担任名・担任メール・年度）の登録・編集・削除
- **生徒管理**：生徒情報の登録・編集・削除、担任は担当クラスのみ操作可
- **保護者管理**：保護者情報・ログイン情報の管理、登録時に初期パスワード自動生成

### 5-5. CSVインポート（新学期の一括登録）

大量のデータを一括登録できます。

| インポート種別 | 用途 |
|---|---|
| 生徒CSVインポート | 新学期の生徒情報を一括登録 |
| 保護者CSVインポート | 保護者アカウントを一括登録 |
| 管理者CSVインポート | 教職員アカウントを一括登録 |

- CSVテンプレートのダウンロード機能付き
- エラー行はスキップし、正常行のみ登録
- インポート後に認証情報（ID・パスワード）を画面表示 → 保護者へ配布

## 6. セキュリティ機能

### ログイン方式

| ユーザー | 認証方式 |
|---|---|
| 管理者（担任・スーパー管理者） | メールアドレス＋パスワード |
| **保護者** | 初期ID・パスワード → **二段階認証（2FA）** |
| 生徒 | Classroomメールで検索し保護者アカウントへ誘導 |

### 二段階認証（保護者ログイン）

1. 初期ID・パスワードでログイン
2. 登録されたメールアドレスに **6桁の認証コード** を自動送信
3. コードを入力して認証完了 → ダッシュボードへ

```
有効期限：10分　／　再送信：60秒のクールダウン　／　使用後即削除
```

### アクセス制御（ロールベース）

- 担任は **自分の担当クラスのデータのみ** 操作可能
- 保護者は **自分の子の欠席連絡のみ** 操作可能
- スーパー管理者のみ全データへアクセス可能

---

## 7. 自動メール通知

保護者が欠席連絡を登録すると、**即座に担任へメールが自動送信**されます。

### メールに含まれる情報

- クラス名　／　出席番号・生徒氏名
- 日付・区分（欠席 / 遅刻）　／　理由
- 登校予定時刻　／　連絡日時

> メール送信が失敗しても欠席連絡の登録はキャンセルされない安全設計です。

---

## 8. 保護者の初回登録フロー

```
① 生徒のClassroomアドレスを入力 → 本人確認
        ↓
② 保護者情報を入力（名前・続柄・電話番号・メール・パスワード）
        ↓
③ 登録完了 → 次回ログインから2FA認証が有効
```

## 9. まとめ

### 「電話の30分」から「24時間デジタル」へ

| 課題（従来） | このシステムの解決策 |
|---|---|
| 8:00〜8:30 しか連絡できない | **いつでも・どこからでも** オンライン連絡 |
| 電話が集中してつながらない | **同時に何件でも** 受け付け可能 |
| 手書きメモによる転達ミス | **自動メールで担任へ即時・確実**に通知 |
| 過去の記録が残らない | **デジタル記録でいつでも確認**可能 |
| 月次集計が手作業 | **リアルタイム統計・月別グラフ**で可視化 |
| 年度初め一括登録の手間 | **CSVインポートで効率化** |
| 不正アクセスのリスク | **二段階認証＋ロールベースアクセス制御** |

### 今後の拡張可能性

- プッシュ通知（LINE / Slack 連携）
- 出欠レポートの PDF 出力
- 複数校対応（マルチテナント化）
